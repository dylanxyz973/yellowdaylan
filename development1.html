JavaScript hobby website

Languages/Frameworks Used:
JavaScript
Bootstrap
Node + Express

/ route (home page)
Display Greeting text

/lists route
Display of completed drawings with each drawing wrapped with a thin light grey border.

Each drawing consists:
• Details button to direct to separate detail page (/details/:id get route) to view drawing details.
• Edit Button and Delete Button
• Its title, description, image, completion date.


/details/:id get route
Upon Clicking details button, users are directed to each drawing with details shown.
Upon Clicking back to drawings button, users are directed back to lists drawing (/lists route).

/deleteItem/:id post route
Upon Clicking delete button, a pop up browser warning delete prompt to confirm deletion.

/updateItem get route
Retrieve the pre-populated fields (data) from drawings:
• Title
• Description
• Completed Date
• Image
• Details

/addItem get route
Retrieve the fields (data) for adding drawings:
• Title
• Description
• Completed Date
• Image
• Details

/addItem post route
Sumbit the require fields (data) and redirect users to /lists route.

Objectives

To create an simple hobby website as part of mini project by applying knowledge of web development by building a fully functional Node.js Express web application. This project requires the ability to handle routing, form submission and display of information using only GET and POST methods with in-memory data

Technical Competencies

Backend Development (Node.js & Express.js)

• Implemented server-side logic using Express.js
• Managed application flow using middleware and route handlers
• Handled HTTP requests and responses using GET and POST methods only

Data Handling (In-Memory Storage)

• Stored and managed drawing data using JavaScript arrays and objects
• Implemented CRUD-like operations without using databases
• Passed data between routes efficiently using route parameters and request bodies

Form Handling & Validation

• Captured user input through HTML forms
• Processed form submissions on the server side
• Ensured required fields were submitted before processing data

Frontend Integration

• Structured web page layout using semantic HTML
• Styled user interface using Bootstrap utility classes

Routing & Navigation Logic

• Designed clear navigation flow between multiple pages
• Used dynamic route parameters to display individual drawing data

Functionalites/Features

Navigation Bar

Toggle between home, drawing list, add drawing

Drawing Management

• Users can create and store new drawing records during a session
• Pre-filled forms when updating existing drawings
• Each drawing entry is displayed with existing format

Dynamic content rendering for detail page

Individual drawing pages show its own respective details

User Interaction

Confirmation prompt is shown before deleting a drawing to prevent accidental removal

Session-Based Data Handling

All data exists during runtime without persistent storage

User Interface

• Simple and organised layout
• Intuitively placed buttons guide users naivgation between pages easily

Building process

Based on technical rules, only node.js and express.js can be used. Databases, REST APIs, or templates (E.g. EJS, Pug, etc) and view engines like EJS cannot be used. Only GET and POST methods can be used for this particular website. Bootstrap must be applied which CSS cannot be applied resulting in constraints of the webpage design. Instead of the standard black navbar, I chose navbar to be yellow because it is my favourite colour. This website consists of 5 main routes which each page serving its different functionalities, its built using GET and POST methods with in-memory data.
