Task Mobile App

Languages/Frameworks Used:
React Native
Expo Go

Objectives

Application of React Native skills to buiild a list task management protoype mobile app.
MyTaskApp is a list task management app built using React Native based on a Expo GO ecosystem.
This app need to utilise the creation of task screens with Create, Read, Update and Delete (CRUD) operations.

Technical Competencies

Mobile Application Development (React Native)

• Developed a mobile application using React Native within the Expo Go ecosystem
• Structured application using functional components

State Management & Application Logic

• Managed application state using React Hooks (useState and useContext)
• Handled task completion toggling logic to update task status dynamically
• Calculated overall task completion statistics including completed and incomplete tasks with percentage results

Context API & Data Flow

• Centralised task data management using TaskContext for global state sharing
• Passed task data and handler functions between screens using props and context
• Ensured consistent data updates across TaskList.js, AddTask.js, EditTask.js and TaskDetails.js screens

Navigation & Screen Management

• Implemented multi-screen navigation using React Navigation
• Enabled navigation between Task List, Add Task, Edit Task, and Task Details screens
• Pre-filled edit form inputs by initialising component state with existing task data passed through navigation parameters

User Interface & Styling

• Designed responsive layouts using React Native StyleSheet
• Applied appropriate margin and padding, font size for improved readability
• Styled task cards with visual indicators for completion status
• Integrated icons for actions such as edit and delete to improve usability
• Styled interactive elements such as buttons, input fields and cards for better user experience

User Interaction & Feedback

• Implemented Alert dialogs to confirm task deletion and display summary information
• Provided visual feedback when tasks are marked as completed or updated
• Enabled real-time search filtering based on task title and description

Application Structure & Project Management Skills

• Organised project files into appropriate folders for components, screens, context and styles
• Applied proper coding naming conventions to components, variables, and styles
• Maintained clear and beginner-friendly code structure with modular logic separation
• Ensured code readability and maintainability throughout the development process

Functionalites/Features

1. Task List Screen
When no tasks are available, it will display a message on the screen.
Otherwise, each task card shows its:
• Title
• Description
• Due date
• Completion status
• Edit button
• Delete icon with a confirmation alert

Tapping on the card toggles its completion state to completed turning the title to a strike-through. Details are not shown on Task Cards. A more details which users can click on it to view details of the tasks which users are directed to a screen called TaskDetails.

At the bottom of the task list screen, there are two main buttons:
• Add Task
• Overall Summary which shows a popup summarising total number of tasks, how many completed and incompleted tasks with the result in percentage form.

2. Add Task Screen
This screen allows users to create new tasks with these fields:
• Title
• Description
• Due date
• Additional details which are optional

3. Edit Task Screen
Upon clicking the edit button in the card, user can update any existing task. The fields are pre-filled with the current information retrieved from any existing information written in the card.

4. Task Details Screen
This screen displays all information about the selected task in a clean layout:
• Title
• Description
• Due date
• Details
• Completion status

5. Search function
There is a search bar, which filters tasks instantly based on title or description.

6. Data Handling
All tasks data are stored in a called TaskContext which ensures that adding, editing, deleting, and toggling tasks works on their respective screens in the app.

Building process

This project given a timeline of 2 weeks to apply to apply key React Native concepts such as list rendering, navigation and state management.

A key learning experience was implementing CRUD functionality in a structured manner. Managing task creation, editing, deletion, and completion toggling required careful handling of state updates to ensure that changes were reflected immediately across the application. Through this process, I improved my understanding of how React Hooks and context work together to maintain synchronised data.

Throughout the app, I used
• Light Pastel Rainbow Gradient backgrounds and Vibrant Pastel Rainbow Gradient Cards to help spark some energy in users when they use the app
• Golden and bronze tones for text and borders
• Silver Rose Navigation header
• Transparent input bars and rounded corners for a refreshing look

This design approach aims to make the task app both functional and visually appealing giving a sense of vibrancy and elegance with a touch of premium app feel.

Overall, this project strengthened my technical skills in React Native while also improving my problem-solving abilities and code organisation practices.

A major limitation of the current implementation is the lack of persistent storage. Tasks are not retained after the application restarts, which reduces real-world practicality. If I had more time, I would further enhance the app by implementing persistent storage (such as AsyncStorage or a database) to retain tasks even after the app is restarted.
