/ route (home page) (index.ejs)
Display Welcome greetings and links to register or log-in

/register route (register.ejs)
Compulsory registration for all new admins and new users with required fields:
• Username
• Email
• Password
• Address
• Contact Number
• Role (dropdown choose user or admin)

If password is shorter than 6 characters, a message appears to inform users that password should be at least 6 or more characters in length.

/login route (login.ejs)
Log-in page for both admins and users requiring these two fields:
• Email
• Password

Log-in and log-out functionality for both users and admins

/shopping route (shopping.ejs) [User]
View available products with its name, price, quantity available, quantity selection (dropdown of up to 5) and the option to buy them.
Clicking the product name allow them to view the product details
Add products to cart by clicking buy products

/cart route (cart.ejs) [User]
See the products that are in the cart with its quantity, total cost and image
Final total is displayed with a back to shopping link to redirect to shopping.ejs

/inventory route (inventory.ejs) [Admin]
Similar to shopping.ejs but with option to either edit products or delete products with a confirmation prompt.


/addProduct route (addProduct.ejs) [Admin]
Add new products to inventory with the following fields:
• Product Name
• Quantity
• Price ($)
• Product Image

Upon clicking Add product button, a new product would be added to bottom which can be viewed on the inventory.ejs (admins) or shopping.ejs (users)

/updateProduct/:id route (updateProduct.ejs) [Admin]
Update (edit) the product details with pre-populated fields from existing product information:
• Product Name
• Quantity
• Price ($)
• Current Image
• Upload New Image

Admins would be redirected to the inventory.ejs upon clicking update product button
Otherwise, if no changes needed to be made admins can click return to inventory button to return to inventory page

Objectives

To design and develop a functional web-based shopping system that allows users to view products and manage a cart. The project aims to support dynamic content rendering, persistent data storage, and full CRUD operations for product management. Additionally, the system is designed to implement role-based access control to clearly separate user and admin functionalities.

Functionalites/Features

Navbar
Role-based feature access, navigation options are displayed based on account role ensuring users and admins only access functionalities relevant to their authentication.

• Click "Logout" which directs back to onboarding page [Users and Admins]
• Click "View Cart" which directs to cart page [Users]
• Click "Add Product" which directs to addProduct page [Admins]

Shopping & Cart Behaviour
The cart dynamically reflects selected products, quantities and total cost. Thus, allowing users to review their selection before returning to the shopping page.

Dynamic Content Display
Product listings and user greetings are based on user account role providing a personalised experience.

Pratical Enhancements
 • shopping.ejs page and inventory.ejs page display Welcome, username (role).
• Buttons and Product on Hover will change colour to enhance attention.
• A pop up browser warning delete prompt to confirm deletion.

Building process

This project was completed within a 2-day timeline and is my first functional website I created.
I focused on implementing core functionalities with a simple and user-friendly design using basic CSS.
Given more time, I would have enhance the visual styling to better resemble a real-world supermarket website and implement additional features.
Something to note: A design consideration is that admin accounts are restricted to admin functionalities only which requires a separate user account to access user features.
