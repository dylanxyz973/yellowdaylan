HealthBro Mobile App Protoype

Application Used:
Figma

Objectives

Design a new mobile application by applying the full UX design process from start to finish with app addressing a real-world problem by demonstrating ability to create effective and user-centered solution while delivering a meaningful and engaging user experience.

Technical Competencies


Colour Theory

• Selected an appropriate and consistent colour scheme
• Ensured colour contrast are WCAG accessibility standards

UI Design Skills

• Designed mobile UI layouts
• Applied design principles
• Intutive flow of elements
• Applied overlay techniques on images to preserve visual readability
• Used column grid and alignment for clean layout structure

UX & Interaction Design

• Created low-fidelity and high-fidelity wireframes
• Implemented user flows and navigation structure
• Developed interactive prototypes with transitions
• Designed layouts according to appropriate mobile UI practices

Component Management (UI Kit)

• Built reusable UI components
• Created variants for different component states
• Ensured visual consistency across screens using shared components

Interactivity

• Mimicked loading states using loading spinners to demonstrate system feedback
• Implemented prototype interactions and transitions
• Simulated realistic mobile app interactions for showcase of interaction

Usability testing

• Conducted basic usability testing using prototype previews
• Identified usability issues and refined designs accordingly

Project Management

• Organised frames, pages, and components systematically
• Renamed layers and components clearly for maintainability

Building process

I chose red-orange (coral) and purple as my colours scheme for this app becauseI coral convey energy, action and motivation while purple provides mindfulness and stability. Together, they complement each other by balancing motivation and calmness and provide a clear visual hierarchy.

I prepared a deck of slides showcasing:
• Problem Defintion (issue statement and product goal)
• Research (qualitative and quantitative questions)
• User Stories, User Journey Maps, User Personas
• Ideation (sitemap and userflows)
• Insights, feedback and suggested improvements
• App Branding

I develop a prototype consist of 5 page showcasing this content:
1. User Sitemap
2. Userflow when using the app
3. UI Kit
4. Low-fidelity prototype
5. High-fidelity prototype

It took two weeks to prepare the slides and find a suitable problem solution. Then followed by developing the prototype with its interactions across the span of 6 weeks.
